Work only on the existing GPOS homepage design.

Very important:
Do not remove any section.
Do not add any new section.
Do not change section order.
Do not change text meaning.
Do not change links, buttons functionality, routes, forms, API calls, business logic, data mapping, animations logic, or component logic.
Only improve the visual design, spacing, background rhythm, containers, cards, typography spacing, and responsive layout.

Goal:
Keep the same current homepage, but make it look like one unified Apple-level premium SaaS website with a clean blue theme.

Theme:
Use a premium blue SaaS style.
Main page background should be a very soft blue/white gradient, not plain separated blocks.
Use these colors consistently:
- Deep navy: #071A3D
- Premium blue: #2563EB
- Light blue: #EFF6FF
- Soft page background: #F8FAFC / #F3F7FF
- Text dark: #0F172A
- Text muted: #64748B
- Border: rgba(37, 99, 235, 0.12)

Global homepage layout rules:
1. Make the whole homepage feel continuous, not like separate stacked containers.
2. Remove unnecessary section-level card wrappers.
3. Do not put every section inside a white rounded box.
4. Keep the main background consistent across the homepage.
5. Only actual content cards should have white backgrounds, borders, radius and soft shadows.
6. Section wrappers should usually be transparent.
7. No random grey/white/blue background switching between sections.
8. Remove irregular blank spaces above and below sections.
9. Keep all sections aligned to the same max-width container.
10. Make spacing consistent from top to bottom.

Container rules:
Use one shared page container width for all homepage sections:
- max-width: 1180px or 1200px
- margin: 0 auto
- desktop horizontal padding: 24px
- mobile horizontal padding: 16px

Avoid nested containers like:
container > rounded section card > inner container > cards

Correct structure should be:
section > page container > content/cards

Spacing rules:
Set consistent vertical spacing:
- Hero section bottom spacing should naturally connect to next section.
- Major sections: padding-top 84px and padding-bottom 84px on desktop.
- Medium sections: 72px top/bottom.
- Mobile sections: 52px to 60px top/bottom.
- Section header to content grid gap: 36px to 44px.
- Heading to paragraph gap: 12px to 16px.
- Remove extra margin-bottom from last child elements inside sections.
- Remove any random mt-20, mb-24, py-32, large empty gaps unless truly needed.

Hero section:
Keep the same hero content and current assets.
Improve only layout and spacing.
Hero should feel premium and open, with a soft blue gradient background.
Do not wrap hero in a heavy card.
Use clean typography:
- heading large, bold, tight line-height
- paragraph max-width around 560px
- buttons aligned cleanly
